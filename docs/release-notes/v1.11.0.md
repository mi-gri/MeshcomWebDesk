# Release Notes – v1.11.0

## Weather API

MeshCom WebDesk now supports **live weather data from external weather platforms** as telemetry fields.

Supported providers:
- **AWEKAS** – Austrian weather network; API endpoint: `https://api.awekas.at/current.php?key=<API-KEY>`
- **Weather Underground (PWS)** – personal weather station network; API endpoint: `https://api.weather.com/v2/pws/observations/current`
- **Simulation** – generates realistic test data without any API key or network access (always available)

### Setup
1. Go to **Settings → 🌤️ Weather API**
2. Select a provider and enter your **Station ID** and **API Key**
3. Configure the **poll interval** (minimum 5 minutes)
4. Save settings – the first fetch starts automatically

### Telemetry integration
Weather data is written directly to the telemetry JSON file using fixed field names.
Use these field names as **JSON-Key** in the **Telemetry Mapping** (Settings → 📡 Telemetry):

| Field | Description |
|---|---|
| `temp_out` | Outdoor temperature |
| `temp_in` | Indoor temperature |
| `humidity_out` | Outdoor humidity |
| `humidity_in` | Indoor humidity |
| `pressure_rel` | Relative air pressure |
| `pressure_abs` | Absolute air pressure |
| `wind_speed` | Wind speed |
| `wind_gust` | Wind gust |
| `wind_dir` | Wind direction |
| `rain_rate` | Rain rate |
| `rain_day` | Daily rain total |
| `uv` | UV index |
| `solar_rad` | Solar radiation (W/m²) |
| `dew_point` | Dew point |

### Bot command
Use `--weather` to query the current weather status via bot:
- Active provider and last measurement values
- Timestamp of last successful fetch
- Error status if the last fetch failed

### Mutual exclusion with HTTP Telemetry API
Weather API and HTTP Telemetry API (`POST /api/telemetry`) write to the **same telemetry JSON file**.
Only one source can be active at a time:
- If **Weather API** is active → HTTP API is disabled and returns `409 Conflict`
- If **HTTP API** is active → Weather API provider selection is locked in Settings

## NET Console – no-password support

The NET Console now connects successfully even when **no password is set** on the MeshCom node.
Previously, a direct `OK` response (no HMAC challenge) was incorrectly treated as an error.

## Bugfixes

- **AWEKAS API key URL encoding** – AWEKAS displays the key URL-encoded in the portal (`%2B`, `%2F`, `%3D`); automatically decoded before the API call via `Uri.UnescapeDataString`
- **Weather API settings persistence** – `WeatherApi` block was missing from `SettingsService` serialization; settings were not saved across restarts
- **Weather API API key decryption** – key was encrypted on save but not decrypted on load; `DecryptMeshcomSettingsPostConfigure` corrected
- **Double encryption prevented** – `aes:`/`dp:` prefix is detected; already encrypted values are not encrypted again
- **API key field empty on load** – password/key fields are never pre-filled; saving with an empty field preserves the existing encrypted value (`EncryptOrKeepExisting`)
- **Build no longer overwrites runtime data** – `data/**` is excluded from build output copy; `appsettings.override.json` and other runtime files survive rebuilds
