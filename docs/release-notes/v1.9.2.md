# Release Notes – v1.9.2

## MH List: Automatic deletion of old entries (hours)

The "MH List: Max. Age" setting has been changed from days to **hours**.
Entries older than the configured number of hours are automatically removed:
- hourly in the background
- immediately when settings are saved
- on application startup from the saved snapshot

Existing values (days) are automatically migrated to hours. `0` disables the feature.

---

## Performance improvements

### Settings page
Section contents are only rendered when opened. On first page load only the 18 section headers are rendered instead of all ~149 bindings – the page opens noticeably faster.

### MH list & map
- New `OnMhChange` event in `ChatService`: the MH list and map now only react to MH-relevant updates, not to every incoming packet.
- Map: marker redraw with 400 ms debounce.
- QRZ.com lookups only for callsigns not yet in the cache.

---

## Responsive status bar

The status bar adapts dynamically to the available width. Elements are progressively hidden when space is insufficient (`ResizeObserver` + `MutationObserver`).

---

## Bugfixes

- **MH Max. Age – persistence**: `MhMaxAgeHours` was reset to 0 after a page change. Bug in `SettingsService` fixed; the value is now correctly saved to `appsettings.override.json`.
