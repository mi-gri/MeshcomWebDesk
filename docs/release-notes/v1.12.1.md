# Release Notes – v1.12.1

## What is new in v1.12.1

---

## 🔄 Automatic Update Check (every 24 hours)

The update check now runs **every 24 hours** while the service is running – not only once at startup.
MeshCom WebDesk runs as a background service and may stay online for weeks; the banner will now appear as soon as a new version is published on GitHub.

---

## 🐳 Docker Image on GitHub Container Registry

Official multi-architecture Docker images are now published automatically on every release:

```sh
docker pull ghcr.io/dh1fr/meshcomwebdesk:latest
docker pull ghcr.io/dh1fr/meshcomwebdesk:v1.12.1
```

Supported platforms: `linux/amd64` and `linux/arm64` (Raspberry Pi 4 / 5).

---

## 🔖 `--version` shows licence status

```sh
MeshcomWebDesk --version
MeshCom WebDesk v1.12.1 – Licensed for DH1FR
```

The version flag now appends the current licence state so the status is visible without starting the full service.

---

## 🔧 Fixes

- **Licence badge showed "unlicensed" after service start** – fixed a race condition where `MainLayout` rendered before `AppLicenseService` invalidated its cache; an `OnLicenseChanged` event now triggers the re-render at the correct moment
- **AI: `auto` sent as model name** – the literal string `auto` was forwarded to the OpenAI API in `GenerateSummaryAsync`, causing `model_not_found`; auto-selection now works correctly in all code paths
