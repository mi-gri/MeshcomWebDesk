# Release Notes – v1.10.0

## Multi-Node Support

MeshCom WebDesk now supports **multiple MeshCom nodes simultaneously**.
Each node has its own UDP connection, callsign, chat tabs, monitor feed and console credentials.

- **Node profiles** are managed under **Settings → Further Nodes**
- Each profile stores: display name, callsign, Device IP/Port, Listen IP/Port, primary flag, TLS certificate fingerprint and TLS password
- **Exactly one node** must be marked as *Primary* – only the primary node drives the MH list, Live Map, beacons, telemetry, bot, OTA and reboot
- A **node switcher** dropdown appears in the Chat header whenever more than one node is configured; switching changes the visible chat tabs and monitor without any UDP reconnect
- **Incoming packets** are routed to the correct node by the sender's IP address – multiple nodes can share the same UDP port (`1799`)
- Chat tabs, monitor messages and state are **saved and restored independently** per node on restart

---

## TLS Console (replaces Telnet)

The new **Console** page (`/telnet`) replaces the classic Telnet connection.
Node serial I/O is now also available **over the network, encrypted via TLS**.

> ⚠️ **Requires MeshCom firmware v4.35p.05.13 or later.**

### Node setup (one-time)
1. Enable TLS Console on the node: `--tlsconsole on`  
   or via the node web interface: **Settings → IP Network Settings → TLS Console**
2. Set a password (recommended): `--passwd <your-password>`

### WebDesk setup
- Enable TLS Console in **Settings → 🖥️ Console**
- Enter Device IP and password (stored encrypted with DPAPI)
- On first connect: click **Trust & Save** to accept and store the TLS certificate fingerprint

### Features
- **Pause / Clear** output
- **OTA Update** – sends `--ota-update` with 5-second countdown dialog; OTA page opens in a new browser tab
- **Reboot** – sends `--reboot` with confirmation dialog
- Each node profile stores its own TLS certificate fingerprint and password

### Serial Console (USB)
- Direct connection via USB/serial (CP210x); COM port and baud rate configurable in Settings
- DTR/RTS held low after `SerialPort.Open()` – the ESP32 does **not** reboot on connect
- Mode (TLS / Serial) switchable without restart

## Multilingual UI – Français, Italiano, Español

The interface is now available in **5 languages**:
🇩🇪 Deutsch · 🇬🇧 English · 🇫🇷 Français · 🇮🇹 Italiano · 🇪🇸 Español

- Select language in **Settings → Language**
- Switching applies **instantly** across all pages – no restart or page reload required
- Language is persisted in `appsettings.override.json`

---

## Beacon interval minimum 8 h

The beacon interval can no longer be set below **8 hours**.
Existing values below 8 h are automatically corrected to 8 h on load.

---

## Telemetry mapping limit

A maximum of **3 telemetry mapping entries** are allowed.
The add button is disabled when the limit is reached.

## Bugfixes

- **Speaker icon** – the speaker toggle in the status bar now persists across chat page reloads (state stored like the bell icon)
- **TLS password** – the encrypted `dp:` string is now correctly decrypted before being sent to the node
- **TelnetEnabled persistence** – setting is correctly written to and read from `appsettings.override.json`; no longer lost after restart or backup restore
- **Multi-node first-connect cert** – `certThumbprintOverride` is passed as `string.Empty` (not `null`) for nodes without a stored fingerprint; prevents Node-1's fingerprint from being incorrectly applied to Node-2
- **Serial connect** – DTR/RTS held low after `SerialPort.Open()`; ESP32 does not reboot on connect
- **Telemetry double-send** – `lastSentSlot` is now initialised to the current hour on startup; prevents a duplicate telemetry send within the first scheduled hour after a restart
- **TLS cert banner** – the *Unknown Certificate* banner is suppressed when Serial Console mode is active
