# Release Notes – v1.12.0

## What is new in v1.12.0

---

## 📅 Calendar Beacon – Scheduled Event Announcements

MeshCom WebDesk can now **automatically send announcements** to a configured group before and/or at a scheduled event time.

- Define recurring or one-time events with a title, group, and event time
- Configure **lead times** (e.g. 60 min and 10 min before the event)
- Preview the **next scheduled occurrence** directly in the settings
- Announcements are sent via the connected MeshCom node – no manual action required

### Setup
1. Go to **Settings → 📅 Calendar Beacon**
2. Add an entry with event title, target group, date/time, and recurrence
3. Configure lead times for advance announcements
4. Save – the beacon scheduler starts automatically

---

## 🖥️ Console Command Helper

The Console Command Helper provides a **graphical assistant** for reading and setting MeshCom node parameters without typing raw console commands.

- Browse available commands grouped by category
- Read current values directly from the node
- Apply changes with a single click – no console knowledge required
- Integrated **mini console popup** for live feedback
- **Spectrum scan chart** for RF visualisation

### Setup
No configuration required. Open the page from the main navigation and connect to your node via Telnet or USB console.

---

## 🔐 Program-wide Licence

MeshCom WebDesk now features a **program-wide RSA-signed licence** bound to the configured callsign (without SSID).

- Unlicensed: header badge shows *Free unlicensed version*
- Licensed: header badge shows *Licensed for \<CALLSIGN\>*
- Licence is validated offline against a signed token
- Future feature gating is supported by this licence model

---

## 🤖 AI / QSO Summary Improvements

- **Auto model selection** – the model is now automatically chosen based on prompt size (`gpt-4o-mini` / `gpt-4.1-mini` / `gpt-4.1`); no manual model selection required
- **Fixed: `auto` sent as model name** – the literal string `auto` was previously forwarded to the OpenAI API, causing `model_not_found`; resolved in both `SearchAsync` and `GenerateSummaryAsync`
- **OpenAI project keys (`sk-proj-`)** are now fully supported
